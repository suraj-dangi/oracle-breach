About the Oracle Cloud Breach

In mid-March 2025, a threat actor publicly claimed to have compromised Oracle Cloud Infrastructure, exfiltrating over 6 million records affecting 140,000+ tenants. The stolen data includes:

Encrypted SSO credentials
Java Keystore (JKS) files
Key files
Oracle Enterprise Manager JPS keys

If you are affected

1. Reset All Credentials

Immediately change passwords and rotate API keys for all Oracle Cloud services and any systems that share credentials.

2. Audit Account Activity

Review Oracle Cloud logs for suspicious activity and check for unauthorized changes to configurations or services.

3. Enable MFA

Implement multi-factor authentication across all privileged accounts if not already enabled.

4. Contact Oracle Support

Report the compromise to Oracle Cloud support and follow their recommended mitigation procedures.
